Use helpers from `@ideative/next-handler/intl` to create translated error variants:

```ts
import { generateTranslatedErrors } from "@ideative/next-handler/intl";

const errors = generateTranslatedErrors({
  notFound: "errors.notFound" as any,
  badRequest: "errors.badRequest" as any,
  forbidden: "errors.forbidden" as any,
  internalServer: "errors.internalServer" as any,
});

throw errors.notFound("User", { id: 42 });
```

These still round-trip through `serializeApiError` / `deserializeApiError`, while carrying `translationKey` and `params`.
